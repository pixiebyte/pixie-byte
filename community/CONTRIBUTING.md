# Contributing to Pixie Byte's Repo

First off — thank you! Whether you're fixing a typo, adding a new cheat sheet, or writing an entire learning path, every contribution makes this resource better for everyone.

## How to contribute

### Quick contributions
- **Found a typo or error?** Open a PR with the fix — no issue needed
- **Have a question?** Open an Issue with the `question` label
- **Want to suggest a topic?** Start a Discussion

### Bigger contributions
1. **Check existing Issues and Discussions** to see if someone's already working on it
2. **Open an Issue** describing what you'd like to contribute
3. **Fork the repo** and create a branch: `git checkout -b feature/your-feature-name`
4. **Make your changes** following the style guidelines below
5. **Submit a PR** linking to the Issue

## Style guidelines

### Writing
- Write for a beginner-to-intermediate audience unless noted otherwise
- Explain jargon when you first use it
- Use practical examples over theory where possible
- Keep a friendly, encouraging tone — no gatekeeping
- British English spelling (colour, defence, etc.)

### Code and scripts
- Include comments explaining what the code does and why
- Add a header comment with: purpose, usage, requirements
- Test your scripts before submitting
- **Defensive tools only** — nothing designed for offensive use without explicit educational framing

### File naming
- Use lowercase with hyphens: `network-scanning-basics.md`
- Cheat sheets: `tool-name-cheat-sheet.md`
- CTF writeups: `platform-challenge-name.md`

## Code of conduct

Be kind. Be helpful. Don't be a jerk. We're all here to learn.

- No harassment, discrimination, or personal attacks
- No sharing of actual exploits, malware, or tools designed for harm
- No gatekeeping — every question is valid
- Credit your sources
- Remember that people of all skill levels read this repo
